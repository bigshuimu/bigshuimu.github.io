commands

ls -l

  1. 文件类型与权限（-rwxr-xr-x）：

    • 第一个字符表示文件类型：

      • -：普通文件
      • d：目录
      • l：符号链接
      • b：块设备
      • c：字符设备
      • p：命名管道
      • s：套接字

    • 接下来的九个字符表示文件的权限，分为三组，每组三个字符，分别代表文件所有者、所属组和其他用户的权限：

      • r：读权限

      • w：写权限

      • x：执行权限

      • -：无权限

  2. 硬链接数（1）： 表示文件或目录的硬链接数量，通常目录会大于 1（目录会包括.和..这个两个），文件通常是 1。

  3. 文件所有者（user）： 表示该文件的所有者。

  4. 文件所属组（group）： 表示文件所属的用户组。

  5. 文件大小（1234）： 以字节为单位显示文件的大小（字节）。

  6. 最后修改时间（Jan 01 12:34）： 表示文件的最后修改时间，格式是月份、日期和时间（若文件是较早修改的，时间可能会替换为年份）。

  7. 文件名（filename）： 文件或目录的名称。

硬链接

ln pp qq 是一个创建硬链接的命令，它的作用是为文件 pp 创建一个新的名称 qq，两个文件共享相同的数据块。

解释：

  • ln：用于创建硬链接或符号链接的命令。
  • pp：源文件，已经存在的文件。
  • qq：目标文件，新的硬链接文件名称。

硬链接的特点：

  1. 文件共享相同的 inode：硬链接文件和源文件指向同一个 inode（文件在文件系统中的数据结构），因此它们实际是同一个文件，只是名称不同。
  2. 数据共享：硬链接文件和原文件的内容是相同的。如果修改 pp 或 qq 中的内容，另一个文件也会同步变化。
  3. 删除其中一个不会影响文件数据：即使删除了 pp 或 qq，只要还有一个硬链接存在，文件数据仍然可以通过剩余的文件名访问

符号链接

  1. 创建一个名为 qq 的符号链接，指向文件或目录 pp。
  2. 访问 qq 时，实际上是在访问 pp。但 qq 只是一个指针，不占用与 pp 相同的硬盘空间。
  3. 如果删除或移动 pp，qq 仍然存在，但它会失效，指向一个不存在的文件。

chown（Change Owner）：

chown 用于更改文件或目录的所有者和/或所属组。

chgrp（Change Group）：

chgrp 用于更改文件或目录的所属组。

显示文件

  • cat：用于连接并显示文件内容，常用于查看文件的全部内容。
  • more：按页显示文件内容，适合浏览长文件，按键盘空格键翻页。
  • less：类似 more，但支持前后滚动浏览文件内容，效率更高。
  • head：显示文件的前几行内容，默认显示前 10 行。
  • tail：显示文件的末尾几行内容，默认显示最后 10 行，可用于监视实时日志。

gdb

启动和退出

  1. gdb <program>: 启动 gdb 并加载程序 <program>。
    • 中文注释: 启动 gdb 并加载程序。
  2. quit: 退出 gdb。
    • 中文注释: 退出 gdb。

运行程序

  1. run 或 r: 运行程序。
    • 中文注释: 运行程序。
  2. start: 运行程序并在 main 函数处暂停。
    • 中文注释: 运行程序并在 main 函数处暂停。

设置断点

  1. break <function> 或 b <function>: 在函数 <function> 处设置断点。
    • 中文注释: 在函数处设置断点。
  2. break <file>:<line>: 在文件 <file> 的第 <line> 行设置断点。
    • 中文注释: 在文件的指定行设置断点。
  3. info breakpoints 或 i b: 显示所有断点信息。
    • 中文注释: 显示所有断点信息。
  4. delete <breakpoint_number>: 删除指定编号的断点。
    • 中文注释: 删除指定编号的断点。
  5. disable breakpoint 3弃用断点
  6. enable 3启用断点

控制程序执行

  1. continue 或 c: 继续执行程序，直到遇到下一个断点。
    • 中文注释: 继续执行程序。
  2. next 或 n: 单步执行，不进入函数调用。
    • 中文注释: 单步执行，不进入函数。
  3. step 或 s: 单步执行，进入函数调用。
    • 中文注释: 单步执行，进入函数。
  4. finish: 继续执行直到当前函数返回。
    • 中文注释: 继续执行直到当前函数返回。
  5. until 或 u: 继续执行直到当前循环或代码块结束。
    • 中文注释: 继续执行直到当前循环或代码块结束。

查看和修改变量

  1. print <variable> 或 p <variable>: 打印变量 <variable> 的值。
    • 中文注释: 打印变量的值。
  2. display <variable>: 每次程序暂停时自动显示变量 <variable> 的值。
    • 中文注释: 自动显示变量的值。
  3. set <variable> = <value>: 修改变量 <variable> 的值为 <value>。
    • 中文注释: 修改变量的值。

查看栈帧

  1. backtrace 或 bt: 显示当前调用栈。
    • 中文注释: 显示调用栈。
  2. frame <frame_number> 或 f <frame_number>: 切换到指定编号的栈帧。
    • 中文注释: 切换到指定栈帧。
  3. up: 切换到上一个栈帧。
    • 中文注释: 切换到上一个栈帧。
  4. down: 切换到下一个栈帧。
    • 中文注释: 切换到下一个栈帧。

查看源代码

  1. list 或 l: 显示当前行的源代码。
    • 中文注释: 显示当前行的源代码。
  2. list <function>: 显示函数 <function> 的源代码。
    • 中文注释: 显示函数的源代码。
  3. list <file>:<line>: 显示文件 <file> 的第 <line> 行附近的源代码。
    • 中文注释: 显示文件指定行的源代码。

其他

  1. info locals: 显示当前栈帧中的局部变量。
    • 中文注释: 显示当前栈帧的局部变量。
  2. info args: 显示当前函数的参数。
    • 中文注释: 显示当前函数的参数。
  3. info registers: 显示寄存器的值。
    • 中文注释: 显示寄存器的值。
  4. help 或 h: 显示帮助信息。
    • 中文注释: 显示帮助信息。

  • 1：标准输出（默认当前终端显示器）
