Rope的前世今生

背景

为什么要引入位置编码

位置编码的作用就是为每个输入token（比如词语）添加一个表示其在序列中位置的向量，使得模型在处理这些token时，能够考虑它们在序列中的顺序。

在经典的Transformer模型中，位置编码和输入词嵌入（word embeddings）相加，确保模型在处理每个token时既考虑词汇的语义信息，也能考虑词汇的位置。

经典位置编码

绝对位置编码：这种方法为每个输入token的位置生成固定的编码，通常使用正弦和余弦函数来生成每个位置的嵌入，且这些编码是固定的。

可学习的绝对位置编码：与绝对位置编码类似，但位置嵌入不是固定的，而是作为模型的可训练参数。这意味着模型可以根据训练数据学习最合适的位置表示。

RoPE（旋转位置编码）：这是一个较新的方法，它通过在复数平面上的旋转来编码位置信息，这样可以更好地处理长序列，并提高模型在不同序列长度上的泛化能力。RoPE在处理长距离依赖关系时比传统的位置编码方法更有效。

RoPE，主要是为了解决传统位置编码的一些限制。例如，传统的绝对位置编码无法有效地处理非常长的序列，尤其是在跨越不同序列长度的任务中表现较差。RoPE通过引入旋转变换，可以更灵活地捕捉位置关系，特别是在跨长距离的依赖中，比传统的固定编码更具优势。

Rope

RoPE 的作用是提供一个函数 ( f )，在计算注意力分数时，将输入的绝对位置 ( m, n ) 代入这个函数，从而让计算出来的注意力分数包含位置信息。这是完全正确的！从数学的角度来说，RoPE 的实现方式确实可以抽象为一个函数 ( f )，它调整了 ( Q ) 和 ( K ) 的计算方式，使得注意力分数能够编码位置信息。

为什么 RoPE 的设计如此高效？

  • 自然编码相对位置信息：RoPE 不需要显式引入相对位置矩阵（例如 T5 的相对位置偏置），而是通过旋转变换将位置信息嵌入 ( Q ) 和 ( K ) 的点积中。
  • 无序列长度限制：传统的正弦位置编码因为是固定维度的编码函数，无法灵活处理超长序列。而 RoPE 的旋转操作基于相位角的累积，可以自然地扩展到任意序列长度。
  • 高效性：RoPE 的计算仅涉及简单的复数乘法（或对应的矩阵乘法），无需引入复杂的额外计算。

补充理解

  1. 复数形式的优雅性：

    • 复数的旋转可以通过指数形式 ( e^{im\theta} ) 直接表达，并且具有简洁的加法与乘法规则，适合推导和实现。
    • 但为了计算机实现，最终都会转化为矩阵操作。

  2. 矩阵形式的必要性：

    • 在实际计算中，矩阵形式更适合数值计算和线性代数框架。
    • 它明确地展示了旋转操作如何作用于二维向量，便于实现。

背景和相关工作

本文的贡献主要体现在三方面：

  • 提出了旋转位置编码（RoPE）方法，通过旋转矩阵将位置编码与上下文表示相结合，改进了相对位置编码的建模。
  • 探讨了RoPE的特性，表明它在相对距离增大时会自然衰减，这对自然语言的建模是有益的，并指出现有的相对位置编码方法与线性自注意力不兼容。
  • 通过在多个长文本数据集上的实验验证了RoFormer模型的优越性，相较于其他方法，RoFormer取得了更好的性能。
